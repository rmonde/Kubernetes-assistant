### 2. `01_test_embedding.py` ‚Äî Embedding Verification
Calls Azure OpenAI `embedding` deployment with a test string and prints the vector length.

**Key pattern:**
```python
client = AzureOpenAI(
    azure_endpoint=os.getenv("AZURE_OPENAI_ENDPOINT"),
    api_version=os.getenv("AZURE_OPENAI_API_VERSION"),
    api_key=os.getenv("AZURE_OPENAI_API_KEY")
)
response = client.embeddings.create(input="test", model="embedding")
print(len(response.data[0].embedding))  # ‚Üí 1536
```

**Result:** `Length: 1536` ‚úÖ

**Key lesson:** `model=` takes the deployment name (`"embedding"`), NOT the underlying model name (`"text-embedding-3-small"`). Azure routes by deployment.

### 3. `02_create_index.py` ‚Äî AI Search Index Creation
Creates the `k8s-knowledge` index with three fields and HNSW vector search.

**Profile chain (must be complete or Azure rejects the index):**
```
SearchField(content_vector)
  ‚Üí vector_search_profile_name="default"
    ‚Üí VectorSearchProfile(name="default")
```

**Key lessons:**
- `vector_search_dimensions=1536` goes on the `SearchField`, NOT on `VectorSearch`
- `Collection(Edm.Single)` is the OData type for a vector (collection of 32-bit floats)
- `SearchIndexClient` = management client (create/delete index); `SearchClient` = data client (upload/search)
- Admin key for write operations; query key for read-only search

### 5. `03_upload_docs.py` ‚Äî Chunking and Indexing
Reads the markdown, splits by `---` into sections, embeds each section, uploads to AI Search.

**Key pattern:**
```python
sections = content.split("---")
doc_id = 0
for section in sections:
    if not section.strip():
        continue
    response = client.embeddings.create(input=section, model="embedding")
    documents.append({
        "id": f"chunk-{doc_id:03d}",
        "content": section.strip(),
        "content_vector": response.data[0].embedding
    })
    doc_id += 1
search_client.upload_documents(documents)
```

**Key lessons:**
- Dimension mismatch fails at upload time per-document, not at index creation
- `statusCode 201` = new document created; `200` = overwritten
- Script is idempotent ‚Äî re-running overwrites with same data, safe to retry
- Free tier Azure AI Search can time out ‚Äî just retry

**Result:** 12 documents, all `status: True`, `statusCode: 201` ‚úÖ

## Interview Questions This Session Covers

**"How do you authenticate to Azure OpenAI from Python?"**
‚Üí `AzureOpenAI(azure_endpoint=..., api_key=..., api_version=...)`. The key goes in `.env`, loaded by `python-dotenv`. Never hardcoded ‚Äî keys in source code get pushed to GitHub and can be harvested by scanners within minutes.

**"Why pass the deployment name as `model=` instead of the model name?"**
‚Üí Azure OpenAI routes API calls to your specific deployment, not the underlying model. Your code is decoupled from model versions ‚Äî when you upgrade from gpt-4o to gpt-4o-turbo, only the deployment changes in Azure, your code doesn't change.

**"What is HNSW and why does Azure AI Search use it?"**
‚Üí Hierarchical Navigable Small World ‚Äî an approximate nearest neighbour algorithm. Exact cosine similarity over 1M+ vectors would require comparing every single vector (too slow). HNSW builds a graph structure at index time that allows sub-linear search ‚Äî you trade a tiny amount of accuracy for massive speed gains.

**"What is the difference between SearchIndexClient and SearchClient?"**
‚Üí `SearchIndexClient` is the management plane ‚Äî create/update/delete index schemas. `SearchClient` is the data plane ‚Äî upload documents and run queries. Same credential can be used for both, but by convention use admin key for management and query key for search.

**"How do you do vector search in Azure AI Search Python SDK?"**
‚Üí `search_client.search(search_text=None, vector_queries=[VectorizedQuery(vector=..., k_nearest_neighbors=3, fields="content_vector")])`. Setting `search_text=None` means pure vector search. You can combine both for hybrid search.
