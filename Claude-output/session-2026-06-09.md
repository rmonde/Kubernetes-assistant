## What Was Covered

### 1. RAG Architecture (First Principles)
**The problem it solves:** GPT-4 knows general K8s concepts but nothing about your team's runbooks, internal docs, or recent incidents. Without RAG it halluccinates or says "I don't know."

**The two-step flow:**
```
User Question
     ↓
[RETRIEVAL] text-embedding-3-small converts question → vector
            AI Search finds top 3-5 most similar doc chunks
     ↓
[GENERATION] gpt-4o reads chunks + question → writes answer
```

The embedding model NEVER generates text. The chat model NEVER touches AI Search. Clean separation.

### 2. Embeddings
- Convert any text → a vector (list of ~1536 numbers) that represents meaning in mathematical space
- Similar meaning = vectors that are geometrically close (measured by cosine similarity)
- Used TWICE: once to embed each doc chunk at index time, once to embed the user question at query time

### 3. Vector Search vs Keyword Search
- Keyword: finds pages containing the exact word "Pod"
- Vector: finds pages semantically similar to "Pod" — containers, workloads, deployments — even with no shared words
- AI Search stores both the original text AND the vector for each chunk

### 4. Azure OpenAI vs OpenAI API
- Same models, but Azure OpenAI keeps data inside your Azure tenant
- Enterprise compliance, VNet support, private endpoints
- Required for regulated industries and enterprise customers

### 5. Model vs Deployment
- **Model** = underlying AI (gpt-4o, text-embedding-3-small) — shared Microsoft infrastructure
- **Deployment** = your named instance with reserved TPM capacity
- Code calls deployment name, not model name — decouples your app from version upgrades
- Pattern: name deployments by role (`embedding`, `chat`) not by model name (`gpt-4o-mini`)

### 6. TPM — Tokens Per Minute
- Capacity unit for Azure OpenAI deployments
- `sku-capacity 50` = 50,000 TPM = ~50 requests/minute at ~1,000 tokens each
- More than enough for a learning project

### 7. versionUpgradeOption
- `OnceNewDefaultVersionAvailable` = Azure auto-upgrades the model when Microsoft sets a new default
- Convenient for dev, risky for prod (behavior can change silently)
- Production teams pin versions and upgrade on their own schedule after testing
