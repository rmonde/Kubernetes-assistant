Four functions wired together in sequence:

```

    → search()          → [chunk1, chunk2, chunk3]

```

## Bugs Fixed This Session

| Bug | Root Cause | Fix |
|---|---|---|
| `credential=os.getenv(...)` | Raw string not accepted by SearchClient | Wrap with `AzureKeyCredential(...)` |
| `field_name=` in VectorizedQuery | Wrong argument name | Use `fields=` (plural) |
| 3 separate chat calls in loop | Loop called build_prompt per chunk | Collect all chunks first, one chat call |
| `search_text=""` | Empty string triggers keyword search | Use `search_text=None` for pure vector |
| join expression outside `{}` | Plain string literal in f-string | Wrap expression in `{}` |
| Model answered out-of-domain questions | No grounding instruction | Added system message + `temperature=0.0` |

## Concepts Covered

**RAG grounding:**
The system message with "Use ONLY the provided context" anchors the model to the knowledge base. Without it, GPT-4o answers from its training data regardless of what context you provide. `temperature=0.0` eliminates randomness for consistent, factual output.

**Why vector search always returns k results:**
`k_nearest_neighbors=3` always returns exactly 3 chunks — there is no concept of "no match." The search engine finds the geometrically closest vectors even if the question is completely unrelated to the knowledge base. The grounding instruction handles irrelevant results gracefully.

**f-string evaluation:**
`f"Context: {"\n\n---\n\n".join(chunk)}"` — the expression inside `{}` is evaluated. Outside `{}` it is treated as a plain string literal. A common mistake.

**AzureKeyCredential:**
Azure SDKs (SearchClient, SearchIndexClient) require the key wrapped in `AzureKeyCredential`, not passed as a raw string. Same pattern as `02_create_index.py` and `03_upload_docs.py`.

**Known enhancement for later:**
Azure AI Search returns `@search.score` with each result. Adding a minimum score threshold would prevent low-relevance chunks from being passed to the LLM at all, making the "I do not know" behaviour more robust.

## Interview Questions This Session Covers

**"How do you prevent a RAG model from hallucinating outside its knowledge base?"**
→ Two layers: (1) system message with explicit instruction "Use ONLY the provided context, say I do not know if not found"; (2) `temperature=0.0` to eliminate creative gap-filling. The retrieval step alone is not enough — the model will still answer from training data if you don't constrain it.

**"What does `search_text=None` do in Azure AI Search?"**
→ Disables keyword/BM25 search entirely. Only vector similarity is used. Setting it to `""` triggers a keyword search on an empty string, which is unnecessary noise alongside the vector query.

**"Why does vector search always return results even for unrelated questions?"**
→ `k_nearest_neighbors` finds the k geometrically closest vectors in the index — it doesn't check whether those vectors are actually relevant. There's no relevance threshold by default. You add one by filtering on `@search.score`.
